4. Installationsanleitung (Schritt für Schritt)

Voraussetzungen

  • Node.js 20 LTS oder höher
  • Git
  • npm (wird mit Node.js mitgeliefert)
  • Supabase-Account (kostenlos unter supabase.com)
  • Stripe-Account (kostenlos unter stripe.com)

Schritt 1: Repository klonen

git clone https://github.com/IHR-USERNAME/praxisonline24.git
cd praxisonline24

Schritt 2: Abhängigkeiten installieren

npm install

Schritt 3: Umgebungsvariablen einrichten

# Vorlage kopieren
cp .env.example .env.local

# .env.local mit eigenen Werten befüllen:
NEXT_PUBLIC_SUPABASE_URL=https://IHR-PROJEKT.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=IHR_ANON_KEY
NEXT_PUBLIC_STRIPE_BASIC_LINK=https://buy.stripe.com/IHR_BASIC_LINK
NEXT_PUBLIC_STRIPE_PRO_LINK=https://buy.stripe.com/IHR_PRO_LINK
NEXT_PUBLIC_STRIPE_BUSINESS_LINK=https://buy.stripe.com/IHR_BUSINESS_LINK
STRIPE_SECRET_KEY=sk_live_XXXXXXXXXX

Schritt 4: Entwicklungsserver starten

npm run dev
# → App läuft auf http://localhost:3000

Schritt 5: Produktions-Build testen

npm run build
npm run start
# → App läuft produktionsbereit auf http://localhost:3000

Schritt 6: Auf Vercel deployen

# Vercel CLI installieren
npm i -g vercel

# Projekt verlinken und deployen
vercel
# oder direkt in Production:
vercel --prod
💡 Wichtig beim Vercel-Deployment: Die Umgebungsvariablen aus .env.local müssen auch im Vercel-Dashboard unter Settings → Environment Variables eingetragen werden.

Supabase-Konfiguration

  1. Neues Projekt unter app.supabase.com erstellen
  2. Unter Settings → API die URL und den anon public key kopieren
  3. Unter Authentication → Email Templates die E-Mail-Templates anpassen
  4. Unter Authentication → URL Configuration die Site-URL auf Ihre Domain setzen

Stripe-Konfiguration

  1. Stripe-Dashboard öffnen → Payment Links
  2. Drei Payment Links erstellen (Basic 39€, Pro 79€, Business 149€)
  3. Die Links als NEXT_PUBLIC_STRIPE_*_LINK in .env.local eintragen
