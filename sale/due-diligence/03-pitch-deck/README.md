# 03 — Pitch Deck

## Purpose

Slide-format presentation of the asset for buyers who prefer a visual walkthrough over long-form text. Investor-standard structure: problem → market → solution → product → traction *(explicitly none, pre-revenue)* → team *(solo founder, offering handover)* → asset → ask.

## Files that will be added

- `pitch-deck.pptx` — Powerpoint version (regenerated via `sale/_build_pptx.py`)
- `pitch-deck.pdf` — flat PDF export of the same content
- `pitch-deck-notes.md` — speaker notes for each slide, if buyer wants to reuse the deck internally
- `slide-source-images/` — high-resolution assets used in the deck

## NDA classification

**Share after NDA.** Contains the same commercial framing as the Investment Report plus screenshot assemblies not visible on the public listing. Provided upon signed NDA.

## Reader notes

- Deck is designed for asynchronous review — every slide is self-contained. No presenter required.
- Total slide count target: **12–15 slides**. If the deck grows past 20 slides, it's off-purpose; split content into the Investment Report instead.
- The "ask" slide is deliberately soft: it names the asking price, transfer terms, and 30-day support commitment. No high-pressure close.
